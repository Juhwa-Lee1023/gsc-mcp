# Gap analysis

## The gap
The open-source GSC MCP landscape still leaves room for a product with all of these traits at once:
- GSC-only
- small tool surface
- read-only by default
- `stdio` first
- explicit accuracy metadata
- planner-aware detail query handling
- property allowlists
- no generic indexing tool
- clear path to mirror and bulk-export routing
- future-ready remote architecture without forcing it into v1

## Positioning statement
Build the “boring but correct” GSC MCP:
- less magical
- less bloated
- more explicit
- more infrastructure-like

## What makes it different
- not a toy wrapper
- not a giant SEO operating system
- not a misleading indexing shortcut
- not a raw pass-through of Google quirks

## Success signal
A cautious engineer should trust it.
A coding agent should find it easy to use.
An SEO should not be misled by incomplete results.
